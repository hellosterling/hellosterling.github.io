Project: Client Document Automation & Data Mapping via n8n

The Problem

Our master list had the same client names repeating across multiple rows. If I ran a basic automation to create docs from that list, it would have created a brand new duplicate document for every single row, wasting cloud storage space and creating a massive manual cleanup chore.

The Solution

I built an automation workflow using n8n to handle the data logic intelligently instead of just running a blind loop:

  • Deduplication: The workflow reads the raw spreadsheet data and filters out the duplicate names, making sure it only processes each unique client once.
  • Document Generation: It automatically references a base template to create exactly one unique Google Doc for each unique client in our folders.
  • Data Mapping: It captures the link of that newly created document and writes it back into every matching row on the master spreadsheet, ensuring no data lines are missed and everything stays perfectly synced.

Core Technical Concepts Practiced

  • Systems Integration: Mapping out the high-level logic in n8n to move data between Google Sheets and the Google Drive API.
  • AI-Driven Code Generation: Utilizing Claude to write the specific data filtering and array formulas needed to isolate unique client names.
  • Prompt Engineering & Troubleshooting: Clearly defining the business rules for the AI, testing its output, and iterating until the data mapped correctly.
